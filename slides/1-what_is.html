What is git?

git is a
distributed version control system
(or DVCS)

Version control system

A version control system
can be seen as the evolution of ctrl+z

The idea is that any state of our project is never lost

At any moment,
we can roll back the project to a previous state

n.b. it's very hard to persuade git
to delete something definitively (luckily)

If you are still versioning using folders,
git is your opportunity to evolve your skills

CLI, but not only

git is usually invoked by command line

There exist many graphical tools,
often integrated in IDEs

Anyway today we will see mostly the CLI usage because graphic tools (mostly) reproduce the same behaviour of the CLI commands
