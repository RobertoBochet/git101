Remote

How to "update" remote repository?

Push

git push origin master

git sends the commits of the master to the remote repository called origin, which tries to merge them to its repository's history.

Remote

And if we want to download the remote commits?

Pull

git pull origin master

git tries to download the commits of branch master
from the remote repository called origin, and it merges them with the local history.

Tries... and if there is a conflict?

Also for a conflict during a push operation we have to be us to handle the conflict

Conflicts must be ever handled by the client-side!

We have to do a pull operation,
solve the conflicts and then do again the push operation.

As a best practices, do a pull operation ever before resuming to work over a project.
