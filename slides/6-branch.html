How to create a branch?

The operation of create a new branch
is called branching

Branch 
git switch -c {branch_name}

To work with branches we generally
use the command switch
With the argument -c, it creates a new branch

n.b. switch is a "new command" (git 2.23) in older guides you may find that checkout is used instead
Branch

Let us create a new branch called "new_branch"

git switch -c new_branch
Switched to a new branch 'new_branch'
n.b. after the creation git will move us to the new branch
Branch

What has happened?

let's check the repository's branches

To do this we can use the command branch which returns the list of the branches in the current repository

git branch
  master
* new_branch

Branch "new_branch" was created and we moved on it

n.b. the char “*” marks the current branch
Branch

How to move to another branch?

git switch {branch_name}
n.b. to move only argument -c is omitted

How to merge two branches?

The merge operation of two branches is called merging

Branch

How to merge two branches?

We have to move to the branch in which
we want to import the changes

git switch {branch_merge_to}

Then we can use the command merge with the argument
the branch whose edits we want to import

git merge {branch_merge_from}

How to merge two branches?

May occur two cases:

  • Between the two branches
    there are no files in conflict
    The merging is successful
  • Some files were edited in both branches
    We must handle the conflicts!
